List — 5 row types × 5 states + leading element axis + section headers

The List has two orthogonal axes: row type (density — 1-line, 2-line, 3-line, multi-line, draggable) and leading element (none, icon, checkbox, radio, thumbnail — the last only valid for 2/3/multi-line rows). The first cell in each row uses real CSS :hover / :active — hover it to verify the natural state matches the forced state next to it. Selected uses brand-blue softer (#EFF5FA), NOT teal per the web colour rule. Section headers use the info-blue num-badge (surface/info/full #1660D9) — this is correctly blue in Figma because it's info-coded, not the brand teal.

In context — four realistic list scenarios

1 · Master-detail sidebar (one row selected)

The selected row gets surface/selected (#EFF5FA brand-blue softer). Last row drops the divider. Master-detail UI: clicking a row updates a detail panel to the right.

2 · Notification settings (checkbox leading)

Settings list with checkbox leading. Bottom row is disabled because the feature requires the mobile app, which isn't installed.

Two section headers group items into "Found" (12) and "Missing" (3). The num-badge is correctly surface/info/full (#1660D9) — info blue, not Scandit brand teal.

4 · Reorderable columns (draggable)

Draggable rows combine the drag handle (drag_handle) AND a checkbox leading element. Common pattern: pick which columns to show + drag to reorder them.
