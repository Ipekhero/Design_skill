Search — 5 states

Single pill-shaped 48px-tall searchbar on a gray surface/formcomp/default surface. State names mirror Figma verbatim: default, hover, selected (keyboard focus + caret + clear button), filled, disabled. The first search bar in the "States" panel uses real CSS :hover and :focus-within — click into it to see the natural selected state match the forced one below.

In context — four realistic searchbar scenarios

1 · Standalone page-level search

Top-of-page filter for the records list. Placeholder hints at the dataset size — useful when the list is long.

2 · With scoping chips above
All Found Missing Pending

Composition: scoping [[chip]] row + the search input. The chip narrows the scope; the search filters inside that scope. Search and chips are independent controls.

Filled state with the clear button visible. Below, the result list updates as the user types (debounce in product code). Clicking ✕ wipes the query and the list resets.
