Text Field — 9 states × 3 sizes × 2 layouts

Single-line input with optional label, mandatory asterisk, helper text, leading icon, and trailing icon button. State names mirror Figma verbatim: default, hover, selected (focused with caret + clear button), filled, disabled, read_only, error, error_hover, error_selected. The first field in the state grid is interactive — click into it to see selected match its forced sibling. Border on focus is near-black (#16191C), NOT brand blue — same call as search.

In context — four realistic forms

1 · Login form (large)

Large size for hero forms. Visibility toggle on the password field. Both fields are mandatory (red asterisk).

2 · Account form with one validation error

Enter a valid email address including the domain extension.

Regular size. The Email field is in the error state — light-red surface, light-red border, red helper text. Validation happens on blur; focusing the field flips it to error_selected with the 2px red border.

3 · Settings panel — small horizontal layout

Compact horizontal layout used inside settings sidebars. "Plan" is read_only (no border, just the value). "Tenant ID" is disabled (border stays but everything mutes).

4 · Scan-into-field (the canonical Figma pattern)

Scan the barcode on the asset tag to fill this field automatically.

The trailing scan button is the canonical Scandit pattern — tap to open the camera and capture a code straight into the field's value. The button is part of the field's slot because its action modifies the field's own value (not a navigation away).
