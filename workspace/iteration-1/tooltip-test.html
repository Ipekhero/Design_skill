Tooltip — top + bottom positions

Brand-blue brief contextual hint that appears on hover or focus of a trigger. Sharp 0-radius rectangle, white text, 16×8 caret pointing at the trigger. Figma defines two positions verbatim: tipPosition: bottom (tip on the bottom of the label, so the tooltip sits ABOVE the trigger pointing down) and tipPosition: top (tip on the top of the label, so the tooltip sits BELOW the trigger pointing up). The first panel shows both forced-visible side by side; the second panel uses real :hover with a 500ms delay — hover or tab onto any trigger and wait half a second.

Short vs long text (max-width clamp)

Short labels fit on one line. Longer labels wrap within the 240px max-width. Anything over ~25 words should move to a callout — tooltips enforce brevity by design.

In context — four realistic tooltip use cases

1 · Icon-only button (label affordance)
Open scanner

The canonical use case — a 32×32 icon button has no visible text, so the tooltip provides the label. The button also carries aria-label="Open scanner" for screen readers; the tooltip is the sighted-user equivalent.

2 · Kebab menu (3-dot vertical)
More actions

A 3-dot vertical kebab — universally recognised but a tooltip removes ambiguity for first-time users.

3 · Unfamiliar terminology / abbreviation
E-12 E-12: device offline — last seen 14:32

An error code badge wrapped in a tooltip that explains what it means + a quick timestamp. cursor: help hints that more info is available on hover.

4 · Help icon next to a setting label

A (i) info icon next to a settings label opens a tooltip explaining the option's behaviour. The tooltip uses tipPosition: bottom here because the icon sits at the top of the visible area — the tooltip needs room ABOVE.
